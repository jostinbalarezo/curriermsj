5. whatsapp_db.py (NUEVO — reemplaza db.py)
--------------------------------------------------------------------------------
- Antes: db.py usaba httpx.AsyncClient (async) y solo accedía a tabla envios
- Ahora: whatsapp_db.py usa httpx.Client (sync, compatible con Flask)
- Funciones:
  - obtener_estado(phone_number) -> Lee estado_usuario de Supabase
  - crear_estado(phone_number, paso, datos) -> Crea registro de estado
  - actualizar_estado(phone_number, paso, datos) -> Actualiza paso y datos
  - reset_estado(phone_number) -> Vuelve a paso "menu"
  - obtener_datos_temp(phone_number) -> Lee JSONB de datos_temp
  - guardar_datos_temp(phone_number, datos) -> Guarda JSONB
  - buscar_faq(pregunta) -> Busca respuesta en tabla faq
  - guardar_reporte(phone_number, descripcion) -> Inserta en reportes
  - guardar_envio(datos) -> Inserta en envios (reemplazo de db.guardar_envio)
  - obtener_envios_por_telefono(phone_number) -> Envíos de un usuario
  - obtener_envio_por_id(envio_id) -> Un envío específico
- datos_temp es JSONB, se serializa/deserializa con json.dumps/loads

6. app.py (NUEVO — servidor Flask + webhook WhatsApp)
--------------------------------------------------------------------------------
- Flask app con 3 rutas:
  - GET /webhook — Verificación de webhook de Meta
  - POST /webhook — Recibe mensajes de WhatsApp
  - GET /health — Health check
- Maneja tipos de mensaje: text, interactive (button_reply, list_reply),
  location, reaction
- Estado almacenado en Supabase (estado_usuario), NO en memoria
- Funciones de envío de mensaje WhatsApp:
  - enviar_mensaje(to, texto) — Texto plano
  - enviar_botones(to, texto, botones) — Botones Reply (max 3)
  - enviar_lista(to, texto, titulo, opciones) — Lista desplegable (max 10)
- Flujos implementados:
  - MENU PRINCIPAL: 5 opciones con botones interactivos
    1. 📦 Rastrear paquete
    2. 🧾 Cotizar envío
    3. 📋 Mis envíos
    4. ❌ Reportar problema
    5. 🧑‍💼 Hablar con agente
  - RASTREAR: Ingresa código CUR-XXXXX, consulta Supabase
  - COTIZAR: Origen (ubicación o texto) -> Destino -> Tipo -> Peso
    -> Cotización estimada con precios mock -> Confirmar envío
  - MIS ENVIOS: Lista envíos del teléfono desde Supabase
  - REPORTAR: Guarda descripción en tabla reportes
  - NUEVO ENVIO: Post-cotización, pide remitente, teléfono,
    destinatario, teléfono dest, dirección destino, tipo paquete,
    instrucciones, muestra resumen, confirma
  - Botón "⬅️ Volver al menú" en todos los pasos
  - FAQ fallback: si no reconoce input en menu, busca en tabla faq
- Cotizaciones mock con diccionario COTIZACIONES (tipo × peso -> precio)
- Tracking code automático: CUR-{ID:05d} (generado por trigger SQL)

7. supabase_schema.sql (NUEVO)
--------------------------------------------------------------------------------
- Tabla estado_usuario:
  phone_number (PK), paso_actual (default 'menu'),
  datos_temp (JSONB default '{}'), updated_at (TIMESTAMPTZ)
- Tabla faq:
  id (SERIAL PK), pregunta, respuesta, categoria (default 'general')
- Tabla reportes:
  id (SERIAL PK), phone_number, descripcion, creado_en (TIMESTAMPTZ)
- Alter tabla envios: agrega tracking_code TEXT, estado TEXT
- Trigger generar_tracking_code: auto-genera CUR-XXXXX al insertar
- RLS habilitado en todas las tablas nuevas
- Políticas: service_role acceso total, anon solo lectura de faq
- Datos iniciales de FAQ: 6 registros (horario, costo, tiempo entrega,
  formas pago, cobertura, abono)
